# 0019 · AI SDK, Cloudflare AI Gateway, and Glass-Box boundary

## Context

DueDateHQ uses AI for mapping messy client data, normalizing tax types, extracting Pulse notices,
briefing dashboard risk, and later answering natural-language compliance questions. These tasks are
high leverage but risky: the product cannot let model output become tax advice, unverifiable
citations, cross-client PII leakage, or request-path latency for the core dashboard.

The stable AI architecture already states "AI SDK only", "No citation, no render",
"Retrieval before generation", "PII never leaves unnecessarily", and "Human-controlled writes".
Dashboard AI Brief later added a sharper product decision: the dashboard request path never waits for
AI; brief output is generated in the background and materialized.

## Decision

All model execution goes through `packages/ai`, Vercel AI SDK Core, and Cloudflare AI Gateway.

- Business modules do not import upstream provider SDKs, provider HTTP clients, or tracing SDKs.
- `packages/ai` owns provider routing, prompt registry access, structured output parsing, usage
  metadata, and model-tier selection.
- Cloudflare AI Gateway is the provider governance layer for caching, retry, rate limiting, and
  provider-level observability.
- DueDateHQ still owns product safety:
  - PII redaction/placeholders before model calls.
  - Retrieval-bounded prompt inputs.
  - Zod schema validation for structured output.
  - Citation validation and out-of-bounds checks.
  - Banned phrase checks for tax-advice-like claims.
  - Internal `ai_output` / `llm_log` trace payloads.
- AI output may propose or explain, but dangerous writes are deterministic server flows triggered by
  explicit user action.
- Dashboard AI Brief is a materialized read model generated by Queue consumers. `dashboard.load` only
  reads deterministic dashboard data and latest `dashboard_brief`; it must not call a model.

## Consequences

Good:

- Provider swaps remain localized to `packages/ai` and deployment configuration.
- The product can audit prompt version, input hash, usage, guard result, and citations.
- Core dashboard SLA is protected from provider latency, cost spikes, and guard rejection.
- AI failure degrades to deterministic UI states instead of blocking the app.

Bad:

- The team must build and maintain guardrails that AI SDK and Gateway do not provide.
- Background materialization adds queue, idempotency, stale-state, and retry complexity.
- Some AI interactions feel less "live" because safety and auditability outrank immediacy.

Uncertain:

- Ask DueDateHQ may need streaming UX later. If enabled, streamed text must remain provisional until
  final guard/citation validation passes.
