DueDateHQ Product thesis
产品判断

先解决的不是提醒，是信任。

风险先于任务

Obligations 不是普通待办。它把截止窗口、预计风险、状态、证据缺口和智能优先级放到同一张表里。

证据先于 AI

AI 不能直接“说服”CPA。每个规则、摘要、导入推断和 Pulse 变更都要挂证据标记或引用来源。

闭环先于通知

Pulse 不是新闻提醒。它必须能从官方源进入复核队列，匹配受影响客户，批量应用，并写入审计与通知记录。

DueDateHQ Client import loop
Migration Copilot

从杂乱客户表到本周分诊，一次导入就进入申报管理。

01 Intake 粘贴、CSV、预设样例和历史同步，都进入同一套导入向导。
02 Mapping AI Mapper 读取字段名和样本行，输出经过契约校验后再进入导入批次。
03 Normalize 实体、州、税种和 EIN 进入归一化；错误行只进入错误列表，不污染最终客户表。
04 Apply + preheat 应用后刷新 Clients、Obligations 和 Dashboard 查询，让新数据马上进入本周分诊。

DueDateHQ AI control boundary
AI 边界

AI 是加速器，不是最终裁判。

无引用，不展示

仪表摘要、obligation 提示和客户风险摘要都必须带来源引用；校验失败就降级拒答。

先检索，再生成

提示词只能引用已检索片段；问答不直连 SQL，而是走自然语言到白名单查询。

写入由人控制

AI 只产出结构化建议；导入应用、Pulse 应用和回滚都由服务端确定性流程加用户确认触发。

AI job Mode Output rule Surface
Migration Mapper / Normalizer fast-json Structured output with trace records Migration Copilot
Pulse Extract quality-json Low confidence routes to manual review Rules > Pulse Changes
Smart Priority explanation deterministic Rule-factor breakdown Dashboard / Obligations
DueDateHQ System architecture
Cloudflare-first 架构

小团队可维护，但边界按生产系统设计。

Vite SPA

apps/app 是登录后产品界面；React Router data mode + TanStack Query + oRPC client。

Cloudflare Worker

apps/server 托管 SPA assets、/rpc、/api/auth、webhooks、health，同一个部署产物。

Native edge services

D1、R2、Vectorize、KV、Queues、Cron 支撑数据、证据、RAG、budget 与后台任务。

Contracts

@duedatehq/contracts 用 Zod + oRPC 共享前后端 API 形状。

Core

@duedatehq/core 保持不依赖基础设施，只承载纯领域逻辑。

DB

Drizzle + D1；结构定义、数据访问、审计和证据写入都集中在 @duedatehq/db。

UI

共享设计变量和基础组件，不让业务功能重复实现表格、状态标记和按钮。

DueDateHQ Product strengths
Operating model

DueDateHQ 把截止日风险接入完整事务闭环。

Obligations control plane

核心界面把状态、风险等级、预计风险和证据放进同一个可追责队列。

Migration activation

导入把旧客户表变成客户和申报义务，并预热 Dashboard 支撑第一周分诊。

AI guardrails

AI 只产出结构化建议和摘要；写入仍由确定性的服务端流程控制。

Production-shaped architecture

Cloudflare-first、oRPC 契约、D1 仓储、租户边界和测试，让系统能解释也能维护。

产品主线很清楚：每个截止日都应该有风险、证据、责任人和审计记录。

Obligations、Migration Copilot 和 Pulse 是同一个可控事务流程的三个部分。
DueDateHQ Closing
Product direction

DueDateHQ 是 CPA 截止日工作的控制层，不是另一个日历。

风险优先

先看截止压力、预计风险、准备度和证据缺口，而不是先翻客户列表。

导入优先

从已有客户表启动事务所，而不是要求团队手工重建基础数据。

规则可复核

把官方源变化变成可复核、可撤销、可审计的截止日更新。

信任内建

租户边界、RBAC、证据和审计链路都放在核心流程里。

DueDateHQ 让截止日工作可观察、可排序、可追责。

这是我为每周都在管理风险的 CPA 事务所做的产品。
